# MedReg Copilot 三分钟演示脚本

## 00:00-00:25 业务入口

打开工作台，说明目标不是替代法规人员，而是把申报材料、法规版本和审核责任放进可追溯流程。进入“申报项目”，展示固定的便携式心电记录仪申请和 42.9% 资料完成率。

## 00:25-01:05 资料预审

进入“资料预审”，展示 3 类已接受资料、缺失类别和最新预审问题。打开一致性检查，指出型号 `RM-ECG-100` 与 `RM-ECG-100A`、心率范围 `30-240 bpm` 与 `30-250 bpm` 的冲突。

## 01:05-01:50 Agent 编制

进入“Agent 编制”，选择风险管理摘要和中英术语受控模式。展示六节点轨迹、法规条款引用、项目证据片段哈希、7 条结构化事实主张和 4 个双语受控术语。强调草稿停留在待人工审批状态。

## 01:50-02:35 评测中心

进入“评测中心”，运行 60 条完整评测。展示数据集版本及 SHA-256、10/10 演示门禁、基线与当前管线对照，以及可筛选的 5 类样本。明确页面显示“待领域专家复核”。

## 02:35-03:00 工程收口

展示架构图和命令 `make demo-up`。说明 PostgreSQL、MinIO、Redis、Qdrant、LangGraph 和 DeepSeek 的边界，并用测试、迁移和可重复评测说明项目不仅是界面 Demo。
