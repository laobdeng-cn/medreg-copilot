# 数据与合规边界

## 可使用数据

- 国家药品监督管理局公开法规、公告、办事指南和指导原则。
- 国家药监局医疗器械唯一标识数据库及公开数据共享接口。
- 明确允许下载和再利用的公开资料。
- 团队自行编写的虚构产品与脱敏申报材料。

## 受限数据

- GB、YY、ISO 等标准全文只有在获得合法授权后才能存储和展示；未授权时仅保存编号、名称、适用范围和合法链接。
- 真实企业申报资料默认视为商业秘密，不进入公开仓库、公共模型训练集或第三方分析服务。
- 临床资料必须使用合成或完成不可逆脱敏的数据；MVP 不处理真实患者身份信息。

## AI 输出边界

- 所有回答必须区分法规原文、系统规则、模型推断和用户决定。
- 找不到证据时返回“证据不足”，不得生成伪引用。
- 模型输出不能直接改变项目状态、关闭问题或签发受控文档。
- 每次生成保存模型、提示词版本、输入文档版本、引用和人工决定。

## 文件入口边界

- 文件在写入 MinIO 前校验扩展名与实际签名；OOXML 同时检查内部包结构、危险路径、加密条目、解压总量、压缩比、宏、嵌入对象和外部工作簿链接。
- HTML 只接受 UTF-8 静态内容并拒绝脚本、iframe、object 与 embed；命中 EICAR 测试签名的内容直接阻断。
- 安全状态、检查器版本、识别类型与发现项随归档记录保存；安全检查上线前的历史归档明确显示为 `legacy`，不伪装为已检查。
- 该确定性检查器只是应用层第一道防线，不等同于完整杀毒引擎。生产环境必须在隔离区增加持续更新的企业级恶意软件扫描、内容净化和人工处置流程。

## 法规图谱边界

- PostgreSQL 法规来源、版本状态和解析证据是事实源；Neo4j 只保存带投影版本的可重建关系副本。
- 只有已核验版本才能把适用、要求和证据关系标记为已核验；关系同时保存生成规则、法规依据、引用标签和原文摘录。
- 版本替代按生效与失效日期生成，上位法引用必须落到可回溯的原文 Chunk；图谱连线本身不能替代法规专业判断。
- 图谱同步属于 Reviewer 权限写操作并进入不可变审计事件流。生产使用前必须由法规专家复核关系全集。

## 租户与访问边界

- 企业申报项目、资料证据、预审结果和 Agent 运行按租户隔离，跨租户查询返回不存在而非泄露资源状态。
- 公开法规与监管原文属于受控共享知识域；新增版本、人工核验和抓取审批仍需法规角色授权。
- Owner、Reviewer、Editor、Viewer 采用最小权限递减；业务载荷中的姓名只作记录，不作为授权依据。
- 创建、归档、复核、预审、Agent 编制、人工决定、评测和报告导出记录操作者、角色、资源、请求与结果。
- MVP 使用可替换的演示身份头解析成员关系；生产部署必须接入企业 SSO/OIDC，并在网关校验签名身份。
