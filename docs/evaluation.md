# M5 评测说明

## 数据集边界

- 版本：`medreg-eval-v1-60`
- 数量：60 条，每类任务 12 条
- 任务：法规检索、引用追溯、跨文档冲突、结构化输出、演示采纳
- 来源：公开法规语义和合成医疗器械注册业务场景
- 状态：合成演示标注已完成，真实医疗器械领域专家签署待完成

数据集保存在 `backend/src/medreg/modules/evaluation/dataset_v1.json`。服务启动时会校验版本、样本数量并计算规范化 JSON 的 SHA-256；运行记录保存该指纹，防止指标与样本版本错配。

## 评分方法

| 任务 | 指标 | 计算方式 |
| --- | --- | --- |
| 检索 | Recall@5、MRR@10 | 相关法规条款是否进入前 5，以及首个相关结果倒数排名 |
| 引用 | 正确率、覆盖率 | 预测引用与金标准引用的微平均交集 |
| 冲突 | Precision、Recall、F1 | 受控字段冲突标签的微平均分类结果 |
| Schema | 通过率 | JSON 有效且包含全部必填结构字段 |
| 采纳 | 演示采纳率 | 合成审阅决策中可进入下一步受控编制的比例 |
| 耗时 | P95 | 60 条样本端到端耗时的最近秩 95 分位数 |

## 可重复运行

```bash
make demo-seed
curl -X POST http://127.0.0.1:8200/api/v1/evaluation/runs \
  -H 'Content-Type: application/json' \
  -d '{"requested_by":"刘凯旗"}'
```

评分引擎不调用外部模型，不依赖随机数。相同代码、数据集版本和比较管线会得到相同质量指标；运行时间戳和运行编号会变化。

## 解释限制

当前数字证明评测链、指标定义和回归门禁可工作，不证明模型已达到生产医疗合规水平。正式上线前应由医疗器械法规专家逐条复核或替换合成标签，并基于真实脱敏材料重新设定阈值。
