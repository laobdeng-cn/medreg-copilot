# 产品规格

## 产品定位

MedReg Copilot 是医疗器械企业内部的注册申报协作平台。它将法规要求、产品资料、证据位置、问题整改和人工决定连接起来，减少资料检索、完整性检查和跨文档核对中的重复劳动。

系统不是医疗诊断工具，也不输出“保证通过注册”等结论。所有生成内容均为草稿，所有预审问题均需要法规事务人员确认。

## 目标用户

- 法规事务专员：建立项目、确定适用法规、组织申报资料和回复补正问题。
- 临床评价人员：维护临床证据、评价路径和引用来源。
- 质量人员：提供设计开发、生产与质量管理体系证据。
- 研发人员：处理产品技术要求、性能指标和风险控制问题。
- 审核负责人：审批 Agent 建议、锁定版本并导出内部预审报告。

## 核心业务流程

1. 创建申报项目，选择 NMPA、器械类别、申报类型和法规生效日期。
2. 系统生成注册资料类别清单，用户分配负责人和截止时间。
3. 上传 PDF、Word、Excel 及中英文资料，解析版面、表格、章节与字段。
4. 根据法规版本和产品上下文执行完整性预审，生成缺口与证据矩阵。
5. 检查名称、型号、适用范围、材料、性能和警示语等跨文档一致性。
6. Agent 生成带原文引用的修改建议或编制草稿。
7. 专业人员接受、修改或驳回，系统保留提示词、输入、输出和决定。
8. 导出内部预审报告、问题整改清单和受控 Word 草稿。

## MVP 范围

### 包含

- NMPA 国内二类、三类医疗器械首次注册项目。
- 法规与指导原则的来源、生效日期、失效日期和替代关系。
- 风险分析、产品技术要求、检验报告、临床评价、说明书与标签、质量管理体系、其他证明资料 7 类清单。
- PDF/Word 文档入库、章节定位和页码引用。
- 混合检索、重排、证据矩阵、确定性规则与 Agent 协作。
- 人工审核、版本快照、操作审计和 Word 报告导出。

### 暂不包含

- 自动判定产品最终分类或是否获批。
- 医疗诊断、治疗建议或真实患者数据处理。
- 未经授权的 GB/YY/ISO 标准全文分发。
- 直接提交国家药监局系统或替代正式电子申报客户端。
- 完整模型训练和私有化 GPU 推理平台。

## 第一条纵向链验收

- 用户可以创建一个 NMPA 二类或三类首次注册项目。
- 必填字段包括产品名称、申请人、类别和法规生效日期。
- 创建后固定生成 7 类资料清单，不依赖 LLM。
- 项目编号唯一，列表和详情返回相同事实。
- 页面能展示资料完成率、待补项和法规基准日期。
- API 输入校验失败时返回明确的 422 响应。
